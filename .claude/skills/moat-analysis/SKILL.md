# 竞争壁垒分析技能 (Moat Analysis)

## 概述

基于巴菲特"护城河"投资理念，提供系统性的竞争壁垒分析框架。本技能**仅负责分析方法论和判断逻辑**，不包含数据获取和报告格式化。

## 职责边界

| 本技能负责 | 本技能不负责（由其他技能完成） |
|-----------|---------------------------|
| 五种壁垒识别与检测方法 | 数据获取（MCP ts-data） |
| 竞争环境评分卡（10分制） | Excel 数据记录生成 |
| 护城河宽度与趋势判断 | Word/DOCX 报告格式化（financial-report-writing） |
| 收入结构拆解分析框架 | 具体的 API 调用流程 |
| 同行对比分析框架 | 图表绘制 |
| 未来3年预测框架 | 具体买卖建议 |

## 数据时效性规则

**分析前必须确认数据截止日期，禁止使用硬编码年份。**

### 动态日期计算

```
根据当前月份确定最新可用财报：

月份 >= 5  → 上一年年报已披露，查近4年年报
月份 < 5   → 上上年年报确定可用

最新季报：
月份 >= 11 → 当年三季报
月份 >= 9  → 当年中报
月份 >= 5  → 当年一季报
月份 < 5   → 上年三季报

示例（2026年2月）：
年报：2021-2024（4年）
最新季报：2025年三季报
```

### 禁止事项
- 禁止硬编码年份（如 `"20201231", "20231231"`）
- 禁止忽略最新季报数据
- 禁止用过时数据得出周期判断结论而不验证最新情况
- 报告中必须注明数据截止日期

## 分析框架（四步法）

### 第一步：收入结构分析

#### 1.1 收入拆解维度

| 维度 | 分析内容 | 所需数据 |
|-----|---------|---------|
| 收入占比 | 各业务板块收入占比变化 | 利润表 + 年报披露 |
| 量 | 销量/产量/用户数增长 | 行业数据 + 公司披露 |
| 价 | 单价/ASP/ARPU变化 | 收入 / 量 |
| 行业对比 | 公司 vs 行业平均 | 同行业公司数据 |

#### 1.2 分析要点
- 拆解"量"和"价"各自贡献了多少增长
- 价格驱动说明有定价权（品牌壁垒信号）
- 量驱动说明有规模优势（成本壁垒信号）
- 与行业增速对比，判断是否跑赢

### 第二步：竞争壁垒识别（巴菲特五种壁垒）

#### 2.1 五种壁垒检测表

| 壁垒类型 | 财务特征 | 检测指标 |
|---------|---------|---------|
| **品牌溢价** | 高毛利率、定价权 | 毛利率 > 行业均值 + 10% |
| **规模优势** | 低成本、高周转 | 营收增速 > 成本增速 |
| **网络效应** | 边际成本递减 | 用户增长 vs 成本增长 |
| **转换成本** | 客户粘性高 | 客户留存率、复购率 |
| **有效规模** | 稳定现金流 | 经营现金流/净利润 > 1 |

#### 2.2 各壁垒详细检测方法

**品牌溢价检测：**
1. 毛利率 vs 行业均值：差值 > 10% 为强品牌
2. 提价能力：过去数年是否成功提价
3. 市场份额：高溢价下份额是否稳定

**转换成本检测：**
1. 客户留存率 > 90% 为高粘性
2. 经常性收入占比 > 70% 为强转换成本
3. 销售费用率低于同行说明获客容易

**网络效应检测：**
1. 用户增长 >> 成本增长
2. 获客成本趋势应下降
3. 市场集中度 CR3 > 70% 为强网络效应

**成本优势检测：**
1. 单位成本趋势是否持续下降
2. 营收增长时成本占比下降
3. 与同行对比单位成本是否最低

**有效规模检测：**
1. 过去5年是否有新进入者
2. 市场份额是否长期稳定
3. 行业产能是否饱和

#### 2.3 壁垒强度评分

对每种识别出的壁垒，按 0-10 分评估：
- 9-10 分：极强，几乎不可突破
- 7-8 分：强，短期内难以被侵蚀
- 5-6 分：中等，存在但有被削弱风险
- 3-4 分：弱，仅提供有限保护
- 0-2 分：不存在或可忽略

### 第三步：竞争环境判断

#### 3.1 竞争环境评分卡（10分制）

| 维度 | 2分 | 1分 | 0分 |
|-----|-----|-----|-----|
| 毛利率趋势 | 上升 | 稳定 | 下降 |
| ROE趋势 | 上升 | 稳定 | 下降 |
| 行业集中度 | 提升 | 稳定 | 分散 |
| 资本开支强度 | 下降 | 稳定 | 上升 |
| 龙头vs二线盈利差 | 扩大 | 稳定 | 收窄 |

**总分评估：**
- 8-10 分：竞争环境显著改善（上推估值）
- 5-7 分：竞争环境稳定
- 0-4 分：竞争环境恶劣（下推估值）

#### 3.2 改善/恶化信号

**改善信号：**
- 龙头份额提升
- 尾部企业退出
- 行业价格稳定或上涨
- 产能利用率提升

**恶化信号：**
- 新进入者增加
- 价格战加剧
- 行业产能过剩
- 技术变革威胁

### 第四步：未来3年预测

基于壁垒强度和竞争环境，预测未来3年：

| 预测维度 | 说明 |
|---------|------|
| 营收增速 | 基于量价拆解 + 行业beta |
| 净利润增速 | 营收增速 + 利润率变化 |
| 净利率趋势 | 壁垒强→利润率稳定或提升 |
| 份额变化 | 竞争改善→份额提升 |

**预测原则：**
- 不做超过3年的远期预测
- 明确列出核心假设
- 给出乐观/中性/悲观三种情景
- 必须附带风险因素

## 护城河宽度判断

### 综合评估矩阵

| 等级 | 定义 | 典型特征 |
|-----|------|---------|
| **宽护城河** | 竞争优势可持续20年+ | 多重壁垒、ROE持续>20%、行业龙头 |
| **窄护城河** | 竞争优势可持续10年左右 | 单一壁垒、ROE 15-20%、有一定地位 |
| **无护城河** | 无可持续竞争优势 | ROE<15%、份额波动、竞争激烈 |

### 护城河趋势判断

| 趋势 | 信号 |
|-----|------|
| **加宽** | 市场份额提升、利润率上升、竞争对手退出 |
| **稳定** | 指标基本稳定、份额维持 |
| **收窄** | 技术变革、新进入者、利润率下滑 |

## 分析输出结构

分析完成后，输出以下结构化结论，供 financial-report-writing 技能格式化为报告：

```
一、公司概况
   - 基本信息（名称、代码、行业、市值）

二、收入结构分析
   2.1 核心财务数据趋势（年报 + 最新季报）
   2.2 量价拆解与增长率分析

三、同行财务对比
   3.1 盈利能力对比表
   3.2 关键发现

四、竞争壁垒识别
   4.1 五种壁垒检测结果
   4.2 核心壁垒详解
   4.3 壁垒强度评分

五、竞争环境判断
   5.1 环境评估评分卡（10分制）
   5.2 改善/恶化信号

六、未来3年预测
   6.1 行业假设
   6.2 公司预测（乐观/中性/悲观）
   6.3 预测依据

七、风险提示

八、结论与总评
   - 护城河类型
   - 护城河宽度（宽/窄/无）
   - 护城河趋势（加宽/稳定/收窄）
   - 竞争环境（改善/稳定/恶化）
```

## 关键财务指标阈值

### ROE 与护城河

| ROE 持续水平 | 护城河判断 |
|------------|-----------|
| >25% | 宽护城河，极强竞争优势 |
| 20-25% | 宽护城河 |
| 15-20% | 窄护城河 |
| 10-15% | 无明显护城河 |
| <10% | 竞争激烈，无护城河 |

### ROE 杜邦分解

```
ROE = 净利率 x 资产周转率 x 权益乘数

护城河来源对应：
- 高净利率 → 品牌溢价、定价权（可持续性高）
- 高周转率 → 成本优势、效率领先（可持续性中）
- 高杠杆   → 金融杠杆（可持续性低，需谨慎）
```

### 综合指标阈值

| 指标 | 优秀 | 良好 | 一般 |
|-----|-----|-----|-----|
| ROE | >20% | 15-20% | <15% |
| 毛利率 | >40% | 25-40% | <25% |
| 净利率 | >15% | 8-15% | <8% |
| 经营现金流/净利润 | >1.2 | 0.8-1.2 | <0.8 |
| 资产负债率 | <40% | 40-60% | >60% |

## 重要原则

### 必须做
- 动态计算查询日期，获取最新可用数据
- 使用真实财务数据，不臆测
- 与同行对比，而非绝对值判断
- 明确假设和风险
- 报告中注明数据截止日期

### 禁止做
- 禁止硬编码年份
- 不基于单一指标下结论
- 不忽视竞争环境变化
- 不做超过3年的远期预测
- 不提供具体买卖建议

## 协作流程

```
MCP ts-data（数据获取）
       ↓ 财务数据
moat-analysis（本技能：分析判断）
       ↓ 结构化分析结论
financial-report-writing（格式化输出）
       ↓
.docx / .xlsx 文件
```
