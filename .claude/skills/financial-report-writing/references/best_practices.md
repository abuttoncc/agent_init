# 报告格式化最佳实践

## 报告结构模板

### 完整报告（含封面）
1. 封面页 — 标题、机构、日期
2. 免责声明
3. 核心提要 — 3-4 个编号要点
4. 一、投资评级 — 三线表
5. 二、核心观点 — 分基本面/技术面/建议
6. 三、市场表现回顾 — 价格统计 + K线数据表

### 简化报告（无封面）
- 省略封面和免责声明，内容连贯不分页

## 内容撰写要点

### 核心提要
- 突出关键数据（ROE、涨跌幅等）
- 简明扼要，3-4 个要点
- 包含明确的投资建议

### 正文写作原则
1. 数据先行 — 先陈述事实数据，后给出分析结论
2. 逻辑清晰 — 使用"首先...其次...最后..."等逻辑连接词
3. 客观中立 — 避免"必然"、"一定"等绝对化表述
4. 风险提示 — 每个投资建议都应附带风险提示

### 数字格式规范
```
价格：保留2位小数     → 32.15元
百分比：保留2位小数   → 3.43%
大数字：使用万/亿单位 → 15.23亿元
```

## 格式检查清单

### Word 文档
- [ ] 页面边距：37/35/28/26mm
- [ ] 标题：黑体三号（16pt）
- [ ] 正文：仿宋四号（14pt），首行缩进 28pt
- [ ] 表格：仿宋小四（12pt），三线表
- [ ] 行距：正文 28pt，小标题 25pt
- [ ] 页脚页码：— N — 格式
- [ ] 数据来源注脚：右对齐

### Excel 图表
- [ ] 坐标轴标题含单位
- [ ] A股颜色规范：红涨绿跌
- [ ] 网格线：仅水平主网格线或隐藏
- [ ] X轴标签间隔合理（不重叠）
- [ ] 数据来源标注

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 只设 `run.font.name` | 同时设 `w:eastAsia` 属性 |
| 直接访问 `cell.paragraph_format` | 通过 `cell.paragraphs[0].paragraph_format` |
| 表格用粗黑网格线 | 三线表：顶底 1.5pt、内部 0.75pt |
| X轴标签堆叠 | 设置 `tickLblSkip` |
| 图表无单位 | 轴标题带单位："价格（元）" |
